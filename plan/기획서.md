> 버전: MVP (Minimum Viable Product)

## 0. MVP 철학

```
"완벽한 기획서 → 개발 → 출시" ❌
"가설 → 빠르게 만듦 → 출시 → 시장이 답해줌" ✅
```

**이 기획서의 모든 내용은 가설이다.**
- 시장 규모? → 추정치
- 타겟 사용자? → 가설
- 필요한 기능? → 예상

**진짜 답은 출시 후 사용자가 알려준다.**
- 100명 모으면 → 진짜 니즈 파악
- 피드백 받으면 → 빠르게 수정
- 안 쓰는 기능 → 과감히 삭제

## 1. 제품 개요

### 1-1. 핵심 가치

**원 라인 피치:**
> "크리에이터의 진짜 수익을 3초 만에"

**핵심 질문:**
> "이번 달 내 통장에 실제로 얼마 남았지?"

**공식:**
```
순수익 = 총 수익 (플랫폼 + 광고) - 총 지출 (인건비 + 기타)
```

**슬로건:**
> "수익부터 지출까지, 크리에이터 사업의 모든 숫자를 한눈에"

### 1-2. 문제 정의 (가설)

크리에이터/스트리머가 겪는 핵심 문제:

| 문제 | 상세 |
|------|------|
| **수익 파편화** | 유튜브, 숲, 치지직 등 수익이 여기저기 흩어져 있음 |
| **수수료 계산 귀찮음** | 별풍선 몇 개 받았는데 실수령액이 얼마지? |
| **지출 관리 부재** | 편집자 고용하면 순수익이 얼마나 남지? |
| **MCN 정산 + 개인 수익 분리** | 유튜브는 MCN이, 스트리밍은 내가 관리 |
| **세금 신고 혼란** | 연말에 자료 정리하느라 며칠 소요 |

### 1-3. 해결책

CreFlow는 다음을 제공:

1. **수익 통합** - 모든 플랫폼 수익을 한 곳에
2. **수수료 자동 계산** - 별풍선/치즈 개수 → 실수령액
3. **인건비 관리** - 협력자(편집자, 썸네일러 등) 지급 기록 (선택)
4. **순수익 계산** - 수익 - 지출 = 실제 내 돈
5. **연간 리포트** - 세금 신고용 PDF 자동 생성

## 2. 타겟 사용자

> 💡 **MVP 마인드**: 완벽한 시장 조사보다 "필요할 것이다"라는 가설로 시작.
> 출시 후 실제 사용자 피드백이 가장 정확한 답을 준다.

### 2-1. 주 타겟 (가설)

**핵심 조건: "본인이 직접 수익을 관리하는 크리에이터"**

| 유형 | 설명 | CreFlow 필요성 |
|------|------|---------------|
| 스트리머 (편집자 없음) | 숲/치지직 위주, 혼자 방송 | ⭐⭐⭐ 수익 통합 |
| 스트리머 → 유튜브 시작 | 편집자 고용 고민 중 | ⭐⭐⭐ 수익 + 지출 |
| MCN 소속 + 멀티플랫폼 | 유튜브는 MCN, 스트리밍은 본인 관리 | ⭐⭐⭐ 스트리밍 수익 |
| 유튜버 + 외주 | 편집자/썸네일러 고용 | ⭐⭐⭐ 순수익 계산 |
| MCN 소속 + 유튜브만 | MCN이 다 해줌 | ⭐ 낮음 |
| 대형 소속사 (버츄얼 등) | 회사가 전부 관리 | ❌ 타겟 아님 |

**공통점:**
- 수익이 여러 곳에 흩어져 있음
- "이번 달 실제로 얼마 벌었지?" 바로 답 못함
- 엑셀/노션으로 관리하다 포기함

## 3. 경쟁 분석

| 도구 | 강점 | 약점 | CreFlow 차별점 |
|------|------|------|----------------|
| **엑셀/노션** | 자유도 | 수동, 까먹음 | 입력만 하면 자동 계산 |
| **토스/뱅샐** | 자동 연동 | 개인 지출 중심 | 사업 수익/지출 분리 |
| **유튜브 스튜디오** | 자동 | 유튜브만 | 멀티플랫폼 + 광고 |
| **삼쩜삼** | 세금 계산 | 1년에 한 번 | 실시간 순수익 |

**CreFlow만의 것:**
1. 순수익 = 수익 - 인건비 (아무도 안 해줌)
2. 광고/협찬 파이프라인 관리
3. 협력자별 지급 관리
4. 세금 신고용 연간 리포트 PDF

**치지직 - 치즈**

| 등급 | 수수료 | 수령률 | 치즈 100,000개 실수령 |

| 프로 | 25%* | **75%*** | ₩75,000 |

> 치즈 1개 = 1원, 환전 시 3.3% 원천징수 추가
> *2025년 12월까지 프로모션 할인 적용 (루키 40%→35%, 프로 30%→25%)

**유튜브**

| 수익 유형 | 수수료 | 크리에이터 수령률 |

#### 수익 입력 방식 2가지

**방식 1: 실수령액 직접 입력**
```
이미 정산받은 금액을 그대로 입력
예: 통장에 찍힌 금액 ₩1,079,400 입력
```

**방식 2: 원본 수치(별풍선/치즈 개수) 입력 → 자동 계산**
```
┌─────────────────────────────────────┐
│  숲 수익 입력                        │
├─────────────────────────────────────┤
│  입력 방식                           │
│  ● 별풍선 개수로 입력                │
│  ○ 실수령액 직접 입력                │
├─────────────────────────────────────┤
│  별풍선 개수                         │
│  ┌───────────────────────────────┐ │
│  │ 15,420                        │ │
│  └───────────────────────────────┘ │
│                                     │
│  💰 예상 실수령액                    │
│  ────────────────────────────────  │
│  별풍선 15,420개 × 110원 = ₩1,696,200│
│  수수료 30% (파트너 BJ) = -₩508,860  │
│  원천징수 3.3% = -₩39,182            │
│  ────────────────────────────────  │
│  실수령액: ₩1,148,158               │
│                                     │
│  [저장하기]                          │
└─────────────────────────────────────┘
```

#### 실수령액 계산 공식

```javascript
// SOOP 별풍선
const rawAmount = count * 110;  // 별풍선 개수 × 110원
const afterCommission = rawAmount * (1 - commissionRate);  // 수수료 차감
const netAmount = afterCommission * (1 - 0.033);  // 원천징수 3.3% 차감

// 치지직 치즈
const rawAmount = count * 1;  // 치즈 개수 × 1원
const afterCommission = rawAmount * (1 - commissionRate);
const netAmount = afterCommission * (1 - 0.033);

// 유튜브 (이미 정산된 금액 입력이 일반적)
// API 연동 시 자동 계산
```

### 4-3. 지출 관리

#### 지출 유형

| 유형 | 설명 | 정산 방식 |
|------|------|-----------|
| 인건비 | 편집자, 썸네일러, 모더레이터, 매니저 | 월정액/건별/수익배분 |
| 장비 | 카메라, 마이크, 조명 등 | 일회성 |
| 구독료 | 어도비, 프리미어, 음원 등 | 월정액 |
| 기타 | 촬영 장소, 소품 등 | 일회성 |

#### 협력자 등록 폼

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| name | text | ✅ | 이름/닉네임 (최대 20자) |
| role | select | ✅ | 편집자/썸네일러/모더레이터/매니저/기타 |
| payment_type | select | ✅ | 월정액/건별/수익배분 |
| payment_amount | number | ✅ | 금액 |
| payment_unit | text | 조건부 | 건별일 때: "원/건" |
| revenue_share_percent | number | 조건부 | 수익배분일 때: 1~100% |
| payment_day | number | ❌ | 정산일 (1~31) |
| memo | text | ❌ | 메모 (최대 200자) |

#### 역할별 기본 설정

| 역할 | 기본 정산 방식 | 기본 금액 예시 |
|------|---------------|---------------|
| 편집자 | 월정액 | 80만원/월 |
| 썸네일러 | 건별 | 2만원/건 |
| 모더레이터 | 월정액 | 30만원/월 |
| 매니저 | 수익배분 | 10% |

#### 지출 자동 계산

**월정액:**
```
이번 달 지출 = payment_amount
예: 편집자 80만원/월 → 80만원
```

**건별:**
```
이번 달 지출 = payment_amount × work_count
예: 썸네일러 2만원/건 × 8건 = 16만원
```

**수익배분:**
```
이번 달 지출 = 이번 달 총 수익 × (revenue_share_percent / 100)
예: 총 수익 500만원 × 10% = 50만원
```

### 4-4. 홈 대시보드

#### 표시 데이터

| 영역 | 데이터 | 계산 방식 |
|------|--------|-----------|
| 메인 | 이번 달 순수익 | 총 수익 - 총 지출 |
| 메인 | 전월 대비 증감률 | (이번 달 - 지난 달) / 지난 달 × 100 |
| 카드 1 | 이번 달 총 수익 | 플랫폼 수익 + 입금완료 광고 |
| 카드 2 | 이번 달 총 지출 | 인건비 + 기타 지출 |
| 일정 | 다가오는 일정 | 마감일/지급일 D-7 이내 |
| 차트 | 6개월 추이 | 월별 순수익 (Phase 2) |

#### 순수익 계산 규칙

```javascript
// 이번 달 총 수익
const totalIncome = 
  platformIncome +  // 플랫폼 수익 (입력된 것)
  adIncome;         // 광고 수익 (status === '입금완료'인 것만)

// 이번 달 총 지출
const totalExpense = 
  collaboratorExpense +  // 협력자 인건비 (status === '지급완료')
  otherExpense;          // 기타 지출

// 순수익
const netIncome = totalIncome - totalExpense;
```

### 8-2. SQL 스키마

```sql
-- 사용자
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email VARCHAR(255) UNIQUE NOT NULL,
  name VARCHAR(100),
  avatar_url TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 플랫폼
CREATE TABLE platforms (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  type VARCHAR(20) NOT NULL,
  tier VARCHAR(20),                  -- 등급: normal/best/partner (SOOP), rookie/pro/partner (치지직)
  commission_rate INTEGER,           -- 수수료율 (%) - 직접 입력 시 사용
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 수익
CREATE TABLE incomes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  type VARCHAR(20) NOT NULL,
  source VARCHAR(20),
  income_type VARCHAR(20),           -- 유튜브: ad/superchat/membership
  input_method VARCHAR(20),          -- direct/raw_count
  
  -- 원본 데이터 (별풍선/치즈 개수 입력 시)
  raw_count INTEGER,                 -- 별풍선/치즈 개수
  raw_amount INTEGER,                -- 원본 금액 (개수 × 단가)
  commission_rate INTEGER,           -- 적용된 수수료율 (%)
  commission_amount INTEGER,         -- 수수료 금액
  withholding_tax INTEGER,           -- 원천징수 금액 (3.3%)
  
  -- 최종 금액
  amount INTEGER NOT NULL,           -- 실수령액
  
  date DATE NOT NULL,
  status VARCHAR(20) DEFAULT 'completed',
  campaign_id UUID REFERENCES campaigns(id),
  memo TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 협력자
CREATE TABLE collaborators (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  name VARCHAR(100) NOT NULL,
  role VARCHAR(30) NOT NULL,
  payment_type VARCHAR(20) NOT NULL,
  payment_amount INTEGER,
  revenue_share_percent INTEGER,
  payment_day INTEGER,
  memo TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 지출
CREATE TABLE expenses (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  collaborator_id UUID REFERENCES collaborators(id),
  category VARCHAR(30),
  description TEXT,
  amount INTEGER NOT NULL,
  work_count INTEGER,
  date DATE NOT NULL,
  status VARCHAR(20) DEFAULT 'paid',
  memo TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 광고/협찬
CREATE TABLE campaigns (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  brand_name VARCHAR(100) NOT NULL,
  amount INTEGER NOT NULL,
  payment_date DATE,
  is_paid BOOLEAN DEFAULT FALSE,
  memo TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 피드백
CREATE TABLE feedbacks (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id),
  type VARCHAR(20) NOT NULL,
  content TEXT NOT NULL,
  email VARCHAR(255),
  created_at TIMESTAMP DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_incomes_user_date ON incomes(user_id, date);
CREATE INDEX idx_expenses_user_date ON expenses(user_id, date);
CREATE INDEX idx_campaigns_user_paid ON campaigns(user_id, is_paid);
```
